本地 ASR 性能与效果测试平台 - 产品需求文档 (PRD)
1. 项目概述
本项目旨在搭建一个前后端分离的本地化自动语音识别（ASR）测试平台。后端基于 Python FastAPI，支持多种开源 ASR 模型的动态加载、卸载与切换；前端基于 Vue 3，提供可视化的调试界面、实时波形展示及性能监控仪表盘。核心目标是帮助开发者高效对比不同模型在实时流式场景下的识别准确率（效果）与资源消耗（性能）。
2. 技术栈规范
•后端：Python + FastAPI（异步高并发框架），负责模型推理、音频预处理、WebSocket 流式通信。
•前端：Vue 3 + Vite + TypeScript，UI 组件库建议采用 Element Plus 或 Ant Design Vue，波形图采用 wavesurfer.js。
•通信协议：HTTP/RESTful（用于模型管理、批量任务提交）、WebSocket（用于实时流式识别推流）。
3. 核心功能模块详述
模块一：系统健康与资源仪表盘
•功能目标：直观展示当前系统的运行状态及硬件资源水位，确保测试环境稳定。
•页面元素与交互：
◦系统状态指示灯：以红绿灯形式展示后端服务是否存活。绿色代表就绪，红色代表异常（如显存溢出、进程崩溃）。
◦GPU/CPU 资源仪表盘：以环形百分比图展示当前 GPU 显存占用率（已用/总共）和 CPU 负载。鼠标悬停时显示具体数值。
◦模型状态看板：以标签云或列表形式展示当前已加载到内存中的模型（如 ）。支持点击“卸载”按钮释放指定模型的显存占用。
◦平均性能趋势：展示最近 10 次识别任务的平均 RTF（实时率）折线迷你图，快速判断系统整体负载情况。
模块二：ASR 实时调试工作台（核心页）
•布局结构：采用上下分层布局。上半部分为输入与控制区，下半部分为结果展示与数据监控区。
•A区：输入与控制层（Top Section）
◦音频上传与波形可视化：支持拖拽上传音频文件（wav/mp3/m4a）。集成交互式波形图，支持缩放、点击跳转播放。若后端开启 VAD（语音活动检测），需在波形图上以高亮色块标出有效人声段。
◦参数配置栏：
▪模型选择下拉框：动态拉取后端已加载的模型列表供用户切换。
▪目标语言选择下拉框：指定本次识别的目标语种（如中文、英文、方言等）。
▪高级选项：包含热词注入输入框（逗号分隔）、ITN（文本规整/数字转换）开关。
▪回放模式控制：提供一个“按真实语速推流”复选框。勾选时，后端严格按音频时长推流文字；不勾选时，后端以最快速度返回结果以提升测试效率。
◦主操作按钮：“开始实时识别”。点击后前端建立 WebSocket 连接，开始传输音频流或模拟流式请求。
•B区：结果与监控层（Bottom Section）
◦左侧：实时转写面板
▪流式文本展示区：文字随着后端 WebSocket 消息逐字/逐词动态渲染。正在识别中的临时文本建议显示为灰色或带下划线，已确定的最终文本显示为黑色。
▪错误日志块：若 WebSocket 异常断开或后端推理报错，在此区域直接渲染红色背景的错误堆栈信息，方便排查问题。
◦右侧：性能数据面板
▪关键指标卡片：实时计算并展示本次任务的 RTF（实时率）、首字延迟、端到端总耗时、累计识别字数。
▪原始报文预览：提供一个可折叠的代码块，展示后端返回的完整 JSON 响应数据，便于开发者核对底层字段。
模块三：批量测试中心
•功能目标：支持一次性上传多个音频文件进行压力测试，并生成对比报告。
•页面元素与交互：
◦批量上传区：支持多文件拖拽上传。
◦任务进度列表：以表格形式展示上传的文件名、当前状态（排队中/识别中/已完成/失败）。每个任务需配备实时进度条（0-100%）。
◦结果导出：提供“导出测试报告”按钮。点击后生成 CSV 或 Excel 文件，内容包含：文件名、最终转写文本、RTF、总耗时、目标语言。注意：导出的报告中不包含字级时间戳，以保持报表简洁。
4. 非功能性需求
•性能要求：单条实时识别的首字延迟应尽可能低，WebSocket 通信需保持长连接稳定，避免频繁断连。
•兼容性：后端需内置音频格式标准化处理（统一转为 16kHz 单声道 WAV），确保无论前端上传何种格式，模型均能正常推理。
•易用性：前端界面需具备清晰的加载状态提示（Loading），在模型加载、音频上传、批量处理期间给予用户明确的反馈。
