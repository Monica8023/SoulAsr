后端 backend/src/engines/：这是为了应对你“支持模型切换”的需求。通过定义一个 base_engine.py 抽象基类，强制要求所有接入的模型（Whisper, FunASR 等）都必须实现标准的 transcribe() 方法。这样在 services/asr_service.py 中，你就可以无感地切换底层模型，代码解耦非常彻底。
后端 backend/src/services/model_manager.py：专门负责模型的加载与释放。你可以在这里维护一个全局的模型字典或单例，结合 FastAPI 的生命周期事件（startup/shutdown），实现对显存/内存资源的精细化控制。
前端 frontend/src/api/：将所有后端接口调用集中在这里管理。当后端接口地址或参数发生变化时，你只需要修改这一个文件夹里的代码，不需要去每个 Vue/React 组件里翻找。
前后端完全分离：前端只负责 UI 交互和数据展示，所有的音频处理、模型推理、指标计算全部放在后端完成。这种架构下，未来如果你想把前端换成小程序或者桌面端应用，后端几乎不需要做任何改动。
