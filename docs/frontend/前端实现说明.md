## 页面说明

### 1. 系统仪表盘 `Dashboard.vue`

- 顶部四张指标卡展示系统状态、GPU 显存占用、CPU 负载和平均 RTF。
- 模型管理区使用 `Element Plus Table` 展示后端可用模型，并允许卸载当前激活模型。
- 右侧图表区包含：
  - `RingChart.vue`：GPU 显存占用环形图
  - `TrendChart.vue`：最近任务 RTF 趋势折线图

当前后端没有单独提供 GPU/CPU/RTF 统计接口，因此 `store/system.ts` 会结合已有模型接口构造占位监控数据。后续如果后端新增 `/metrics` 或 `/dashboard` 接口，只需替换该 store 的数据源。

### 2. 实时调试工作台 `DebugWorkbench.vue`

- 上半部分：
  - 使用 `Element Plus Upload` 实现拖拽上传。
  - 使用 `WaveformPlayer.vue` 集成 `wavesurfer.js` 渲染波形图，并预留 `vadSegments` 区域高亮。
  - 提供模型、语言、热词、ITN、实时节流等参数。
  - “开始实时识别”按钮会发起 WebSocket 连接。
- 下半部分：
  - 左侧展示最终转写和临时转写。
  - 识别失败或连接错误时，直接在下方展示红色错误日志块。
  - 右侧展示 RTF、首字延迟、总耗时、处理延迟，并支持原始 JSON 报文折叠预览。

### 3. 批量测试中心 `BatchTest.vue`

- 使用 `Element Plus Table` 展示文件名、模型、状态、进度、RTF、总时长。
- 使用 `FileSaver.js` 导出 CSV，不包含时间戳字段。
- 当前数据由 `store/batch.ts` 生成示例任务，后续可替换为后端批量任务接口。

## 核心封装

### Axios `src/api/client.ts`

- 统一处理 `baseURL`，默认读取 `VITE_API_BASE_URL`，未配置时回退到 `http://localhost:8000/api/v1`。
- 响应拦截器会识别后端统一结构 `{ code, message, data }`。
- 当 `code !== 0` 或 HTTP 错误时，直接用 `Element Plus` 消息提示错误。

### WebSocket Hook `src/composables/useWebSocket.ts`

- 封装连接、发送、关闭、错误状态与断线重连。
- 支持发送 JSON 和二进制数据。
- 在 `DebugWorkbench.vue` 中，推流流程为：
  1. 发送 `start` JSON 帧
  2. 分块发送音频文件二进制
  3. 发送 `end` JSON 帧

当前默认 WebSocket 地址读取 `VITE_ASR_WS_URL`，未配置时回退到 `ws://localhost:8000/ws/asr`。

## 已实现的环境变量约定

```bash
VITE_API_BASE_URL=http://localhost:8000/api/v1
VITE_ASR_WS_URL=ws://localhost:8000/ws/asr
```
