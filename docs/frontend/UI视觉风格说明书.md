### ASR 测试平台 - UI 视觉风格说明书

#### 1. 整体设计语言：赛博科技风 (Cyber-Tech)

- **核心理念**：采用深色模式为基底，结合高饱和度的霓虹色点缀，营造沉浸式的数据监控氛围。打破传统后台管理系统的死板网格，引入悬浮感、透明度和微动效。
- **背景基调**：拒绝纯黑，使用带有微妙径向渐变的深蓝/深灰背景，模拟深邃的科技空间感。

#### 2. 色彩体系规范

- **全局背景色**：`#0F172A` (深邃蓝灰) 或 `#0B0E14` (极夜黑)。建议使用 CSS 径向渐变，中心略亮，四角暗化。
- **卡片/模块背景**：`rgba(30, 41, 59, 0.7)`。必须配合 **毛玻璃效果 (Glassmorphism)**，即添加背景模糊滤镜 (`backdrop-filter: blur(12px)`) 和极细的半透明白色边框 (`border: 1px solid rgba(255, 255, 255, 0.1)`)。
- **主色调 (Primary / 科技感)**：`#00F0FF` (赛博青) 或 `#3B82F6` (电光蓝)。用于主要按钮、激活状态、波形图主体。
- **辅助色 (Secondary / 灵动与活力)**：`#7C3AED` (霓虹紫) 到 `#EC4899` (荧光粉) 的渐变色。用于图表中的次级数据、标签云、以及强调“加载中”的动态效果。
- **功能色**：
    - **成功/健康**：`#10B981` (荧光绿)，带外发光效果。
    - **警告/异常**：`#F59E0B` (琥珀橙) 或 `#EF4444` (警示红)，需配合呼吸灯动画。
- **文字颜色**：标题使用纯白 `#FFFFFF`，正文使用浅灰 `#CBD5E1`，禁用或次要信息使用暗灰 `#64748B`。

#### 3. 核心组件视觉细节

- **仪表盘 (Dashboard)**：
    - **卡片设计**：所有指标卡片采用圆角矩形 (`border-radius: 16px`)，悬停时轻微上浮并增加阴影 (`box-shadow: 0 0 15px rgba(0, 240, 255, 0.2)`)。
    - **图表样式**：ECharts 图表去除多余的网格线和坐标轴，保留极简线条。环形图采用渐变色描边，并在中心显示大号数字。折线图下方需填充半透明的渐变色块，营造“能量流动”的感觉。
- **波形图 (Waveform)**：
    - **配色**：已播放部分使用主色调（赛博青），未播放部分使用低透明度的灰色。
    - **指针**：播放指针设计为一条发光的垂直线，顶部带有三角形指示标。
    - **VAD 高亮**：有效人声段使用半透明的紫色 (`rgba(124, 58, 237, 0.3)`) 覆盖在波形上方。
- **实时转写面板**：
    - **文本展示**：使用等宽字体或现代无衬线字体（如 `Inter`, `Roboto Mono`）。
    - **流式特效**：正在识别的临时文字显示为“打字机闪烁光标”效果；已确定的文字逐渐从半透明变为完全不透明。
- **按钮与交互控件**：
    - **主按钮**：使用线性渐变背景（青到蓝），无边框，添加内发光。点击时有明显的缩放反馈。
    - **开关 (Toggle)**：设计成胶囊形状，开启状态下内部滑块带辉光。

#### 4. 动效与交互体验 (灵动感来源)

- **入场动画**：页面加载时，各个卡片不要同时出现，而是采用交错淡入向上浮动 (`fade-in-up`) 的效果，延迟间隔约 0.1s。
- **数据呼吸感**：系统状态的红绿灯、GPU 占用率数值，在更新时添加柔和的颜色过渡或轻微的缩放脉冲动画，让数据看起来是“活”的。
- **流光边框**：对于当前正在工作的模块（例如正在识别的调试台），可以在卡片边缘添加一圈缓慢流动的渐变光效，提示用户系统正在活跃运行。
- **鼠标跟随**：在空白背景区域，可以添加极其微弱的粒子效果或光晕，随鼠标移动产生轻微的排斥或吸引互动（可选，增加高级感）。

#### 5. 字体与排版

- **数字字体**：所有的性能指标（RTF、耗时、显存）强制使用等宽数字字体（如 `JetBrains Mono` 或 `DIN Alternate`），确保数字跳动时不会左右抖动，强化精密仪器的感觉。
- **层级区分**：通过字重（Bold/Medium）和透明度来区分信息层级，而不是单纯靠缩小字号，保证界面的通透性。
