### 推荐的前端代码生成提示词

**【角色设定】**
你是一位精通 Vue 3 + TypeScript 的高级前端工程师。请基于以下产品需求文档（PRD）和技术规范，为我搭建一个本地 ASR（自动语音识别）性能与效果测试平台的前端项目。

**【技术栈要求】**

- **核心框架**：Vue 3 (Composition API, `<script setup>`) + Vite + TypeScript
- **UI 组件库**：Element Plus（需配置按需引入或全局样式）
- **状态管理**：Pinia
- **路由管理**：Vue Router
- **网络请求**：Axios（封装拦截器，处理基础 URL 和错误响应）
- **可视化图表**：ECharts（用于仪表盘环形图/折线图）
- **音频波形**：wavesurfer.js（用于展示音频波形及交互）
- **工具库**：Day.js（时间格式化）、FileSaver.js（文件导出）

**【页面结构与功能需求】**

**1. 整体布局 (Layout)**

- 采用经典的后台管理系统布局：左侧为固定侧边栏导航，右侧为内容展示区。
- 侧边栏菜单包含两个核心路由：“系统仪表盘” 和 “ASR 调试工作台”。

**2. 页面一：系统健康与资源仪表盘 (Dashboard.vue)**

- **顶部卡片组**：展示“系统状态”（使用红绿灯 Badge 徽章）、“GPU显存占用”、“CPU负载”、“当前平均RTF”。
- **模型管理区域**：以表格或卡片形式列出后端已加载的模型，显示模型名称及状态（Active/Idle），并提供“卸载模型”的操作按钮。
- **图表展示**：使用 ECharts 绘制 GPU 显存占用的环形图，以及最近任务 RTF 趋势的迷你折线图。

**3. 页面二：ASR 实时调试工作台 (DebugWorkbench.vue)**

- **布局方式**：严格采用上下分层布局。
- **上半部分（输入与控制区）**：
    - **文件上传**：支持拖拽上传音频文件，上传后解析并在下方渲染波形图。
    - **波形图交互**：集成 wavesurfer.js，支持缩放、点击跳转播放。预留 VAD 高亮色块的接口。
    - **参数配置栏**：包含“模型选择下拉框”、“目标语言下拉框”、“热词注入输入框”、“ITN开关”，以及一个控制流式输出速度的“按真实语速推流”复选框。
    - **主按钮**：醒目的“开始实时识别”按钮，点击后触发 WebSocket 连接。
- **下半部分（结果与监控区）**：左右分栏。
    - **左侧（转写面板）**：实时展示 WebSocket 推送的文字流（区分临时结果和最终结果）。若发生错误，在该区域下方直接渲染红色背景的错误日志块。
    - **右侧（性能数据）**：展示本次任务的 RTF、首字延迟、总耗时等关键指标，并附带一个可折叠的“原始 JSON 报文预览”代码块。

**4. 页面三：批量测试中心 (BatchTest.vue)**

- **任务列表**：使用 Element Plus 的 Table 组件，展示文件名、状态、带百分比的进度条（Progress）。
- **操作区**：提供“导出测试报告”按钮，点击后将当前列表数据导出为 CSV 或 Excel 文件（不包含时间戳字段）。

**【交互与逻辑细节】**

- **WebSocket 通信**：请封装一个自定义 Hook (useWebSocket) 用于处理实时音频流的发送与文本结果的接收，需包含断线重连和错误捕获机制。
- **响应式设计**：界面需适配常见的桌面端分辨率，保证波形图和图表在不同屏幕下正常缩放。
- **代码规范**：请将页面拆分为独立的 Vue 单文件组件，提取可复用的 UI 小组件（如 MetricCard、WaveformPlayer），并保持代码的高可读性和类型安全。

**【执行步骤】**

1. 请先给出项目的目录结构树。
2. 依次输出核心页面的 Vue 代码（Dashboard.vue, DebugWorkbench.vue, BatchTest.vue）。
3. 输出 Axios 和 WebSocket 的核心封装代码。

---

### 提示词设计思路解析

1. **明确技术栈**：指定了 Vue 3、TypeScript、Element Plus 和 Pinia，避免 AI 随机选用你不熟悉的库（比如选了 React 或 Ant Design）。
2. **细化页面布局**：特别强调了“上下分层布局”和“左右分栏”，确保生成的 UI 结构完全符合你之前的决策。
3. **强调核心交互**：明确指出了 `wavesurfer.js` 和 `WebSocket` 的处理要求，这是实时 ASR 测试平台最复杂的部分，提前告知 AI 可以减少后续的返工。
4. **拆分执行步骤**：让 AI 先出目录结构再出具体代码，可以防止单次回复篇幅过长导致代码截断，也能让你先确认整体架构是否合理。
