### 推荐的后端代码生成提示词

**【角色设定】**
你是一位精通 Python FastAPI 的高级后端工程师。请基于以下产品需求文档（PRD）和技术规范，为我搭建一个本地 ASR（自动语音识别）性能与效果测试平台的后端项目。

**【技术栈要求】**

- **核心框架**：FastAPI（利用其原生异步特性处理高并发推理请求）
- **数据校验**：Pydantic（定义严格的接口请求体与响应模型）
- **音频处理**：librosa 或 pydub（用于音频格式标准化、重采样）、funasr 自带的 VAD 模块（用于语音活动检测）
- **系统监控**：psutil 和 pynvml（用于获取 CPU、内存及 NVIDIA GPU 的显存占用率）
- **并发与队列**：asyncio（处理流式任务）、Background Tasks（处理批量异步任务）
- **通信协议**：WebSocket（实时流式推流）、HTTP/RESTful（常规业务接口）

**【项目架构与目录结构】**
请严格遵循分层架构设计，确保代码的高内聚低耦合：

1. **路由层 (api/)**：负责接收前端请求，进行参数校验并返回标准响应。需包含版本控制（如 `/api/v1`）。
2. **服务层 (services/)**：处理核心业务逻辑，包括音频预处理（VAD切分、格式转换）、模型管理器调用、以及性能指标（RTF、延迟）的计算。
3. **推理引擎层 (engines/)**：封装不同 ASR 模型的统一接口。定义抽象基类 `BaseEngine`，强制子类（如 `WhisperEngine`, `FunASREngine`）实现标准的 `transcribe()` 方法，方便后续动态切换模型。
4. **模型管理层 (model_manager.py)**：作为全局单例，负责模型的动态加载、卸载、缓存管理，以及实时监控硬件资源状态。

**【核心功能模块详述】**

**1. 模型管理与健康检查接口**

- **模型动态加载/卸载**：提供 POST 和 DELETE 接口，支持根据模型名称动态将模型加载到显存或从显存中释放。
- **系统健康检查 (Health Check)**：提供 GET 接口，返回系统整体状态（红绿灯标识）、当前已加载的模型列表、实时的 GPU 显存占用率（已用/总共）以及 CPU 负载情况。

**2. ASR 实时流式识别接口 (WebSocket)**

- **流式通信**：建立 WebSocket 连接，接收前端传来的音频流或模拟流式请求。
- **参数支持**：在握手或初始消息中接收目标语言、热词注入列表、ITN（文本规整）开关等配置。
- **回放模式控制**：根据前端传入的“按真实语速推流”布尔值，决定后端是以最快速度返回识别结果，还是严格按照音频的真实时长进行延时推流。
- **实时数据推送**：通过 WebSocket 持续向前端推送带有临时/最终标记的转写文本，以及实时的性能指标（首字延迟、当前 RTF）。

**3. 批量测试与异步任务接口**

- **批量上传与处理**：提供接受多文件上传的 POST 接口。接收到文件后，将其放入后台任务队列中进行异步处理，避免阻塞主线程，并立即返回唯一的任务 ID (Task ID)。
- **任务进度查询**：提供根据 Task ID 查询处理进度的 GET 接口，返回当前状态（排队中/识别中/已完成/失败）及百分比进度。
- **结果导出支持**：当任务完成后，提供获取完整结果的接口。返回的数据需包含文件名、最终转写文本、总耗时、RTF 等字段，以便前端生成不包含时间戳的简洁 CSV/Excel 报告。

**【非功能性需求与交互细节】**

- **音频标准化**：在后端内置鲁棒的音频预处理逻辑，无论前端上传何种格式（mp3, m4a, flac 等），均统一转换为模型要求的标准格式（如 16kHz 单声道 WAV）。
- **错误处理**：在 WebSocket 断连、模型推理崩溃或显存溢出时，捕获异常并通过特定的错误码或日志块格式返回给前端，方便排查问题。
- **性能指标计算**：精确计算并返回 RTF（实时率 = 推理耗时 / 音频时长）、端到端延迟等关键性能数据。

**【执行步骤】**

1. 请先给出符合上述分层架构的后端项目目录结构树。
2. 依次输出核心模块的代码：包括 `main.py` 启动入口、`model_manager.py` 模型管理器、`base_engine.py` 及其具体模型实现、以及核心的 `asr_service.py` 和路由接口代码。
3. 输出 Pydantic 的核心数据结构定义（Schemas）。
